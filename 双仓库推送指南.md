# 双仓库推送指南（公共 GitHub + 私有 Gitee）

## 远程仓库信息

- 公共仓库（`public`）：`https://github.com/ScottZt/jin-ce-zhi-suan.git`
- 私有仓库（`private`）：`https://gitee.com/SeniorAgentTeam/jin-ce-zhi-suan.git`

## 分支约定

- `main`：公共开源分支（不包含私密配置）
- `private-main`：私有分支（可包含 `config.private.json`）

## 一次性初始化（如已配置可跳过）

```bash
git remote rename origin public
git remote add private https://gitee.com/SeniorAgentTeam/jin-ce-zhi-suan.git
git remote set-url public https://github.com/ScottZt/jin-ce-zhi-suan.git
git branch private-main main
```

如果执行 `git remote rename origin public` 报错 `No such remote: 'origin'`，说明当前仓库没有 `origin`，请直接执行：

```bash
git remote add public https://github.com/ScottZt/jin-ce-zhi-suan.git
git remote set-url public https://github.com/ScottZt/jin-ce-zhi-suan.git
git remote add private https://gitee.com/SeniorAgentTeam/jin-ce-zhi-suan.git
git remote set-url private https://gitee.com/SeniorAgentTeam/jin-ce-zhi-suan.git
git branch private-main main
```

## 一键推送脚本（推荐）

脚本文件：`scripts/push_dual_repo.ps1`

### 1) 先演练（不真正推送）

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -DryRun
```

### 2) 默认模式：自动提交 + 推送公共与私有

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1"
```

默认会：

1. 在 `main` 执行 `add/commit/push public main`（并自动移除 `config.private.json` 的跟踪）
2. 切到 `private-main` 合并 `main`
3. 如存在 `config.private.json`，执行 `git add -f config.private.json`
4. 在 `private-main` 执行 `commit/push private private-main`

### 3) 本地手工提交，脚本仅负责推送

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -UseLocalCommits
```

说明：

- 启用 `-UseLocalCommits` 后，脚本不会自动 `git add` / `git commit`
- 若检测到未提交改动会直接中止，避免把提交控制权从你本地拿走

### 4) 仅推私有仓库（不推公共）

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -SkipPublicPush
```

如果希望“本地提交 + 只推私有”：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -UseLocalCommits -SkipPublicPush
```

### 5) 仅推公共仓库（不推私有）

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -SkipPrivatePush
```

### 6) 自定义提交信息（自动提交模式下生效）

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "scripts/push_dual_repo.ps1" -PublicCommitMessage "feat: your message" -PrivateCommitMessage "chore: update private config"
```

### 7) 参数冲突说明

- `-SkipPublicPush` 与 `-SkipPrivatePush` 不能同时使用

## 手工命令（备用）

### 推送公共仓库（GitHub）

```bash
git checkout main
git add -A
git rm --cached --ignore-unmatch config.private.json
git commit -m "feat: your message"
git push public main
```

说明：`config.private.json` 不存在或未被 Git 跟踪时，上面的命令会直接跳过，不会报错。

### 推送私有仓库（Gitee）

```bash
git checkout private-main
git merge main --no-edit
git add -f config.private.json
git commit -m "chore: update private config"
git push private private-main
```

## 日常建议流程

1. 先在 `main` 开发并推送到 `public/main`
2. 切到 `private-main` 合并 `main`
3. 私有配置有变化时再提交 `config.private.json`
4. 推送到 `private/private-main`

## 安全注意事项

- `config.private.json` 已被 `.gitignore` 忽略，提交私有分支时必须使用 `git add -f`
- 不要把 `private-main` 反向合并到 `main`
- 密钥仅保存在 `config.private.json` 或环境变量，不写入 `config.json`
