## 4) 全局流程接入点

1. 回测启动前按 `strategy_profile_mapping` 自动选择 `profile`。
2. 将 `profile.cost_model` 映射到系统已有字段：
   - `trading_cost.commission_rate`
   - `trading_cost.min_commission`
   - `trading_cost.stamp_duty`
   - `execution.slippage`
3. 将执行约束映射到交易规则校验链路，保持涨跌停与停牌不可成交。
4. 实盘阶段启用 `live_monitoring`，每日生成偏差报告、每周生成归因报告。
5. 未声明策略标签时使用 `fallback_profile`。

## 5) 与当前系统字段对齐关系

- `cost_model.commission_rate` -> `trading_cost.commission_rate`
- `cost_model.min_commission` -> `trading_cost.min_commission`
- `cost_model.stamp_duty_sell` -> `trading_cost.stamp_duty`
- `cost_model.slippage` -> `execution.slippage`
- `risk_alerts.daily_drawdown_critical` -> `risk_control.max_daily_loss_pct`
- `risk_alerts.single_position_weight_critical` -> `risk_control.max_pos_per_stock`
- `risk_alerts.max_drawdown_critical` -> `risk_control.max_drawdown_pct`

## 6) 落地建议

- 默认数据源固定为 `default`，回测阶段不切换。
- 策略优化保持通用化，不做单票定制拟合。
- 回测最短窗口建议 5 年，资金曲线评估建议 10 年。
- 新策略先走“回测 -> 模拟盘 -> 小资金实盘”三级准入。
