# 批量回测操作指南（含覆盖率硬门禁）

本文档用于快速完成以下目标：
- 准备策略池、标的池、区间池、场景池
- 自动生成批量回测任务
- 执行覆盖率检查与硬门禁
- 通过门禁后执行批量回测（串行或并发）

---

## 1. 前置条件

- 进入项目根目录：`D:\04.量化\jin-ce-zhi-suan`
- 本地服务可访问：`http://127.0.0.1:8000`
- 使用 Python 运行脚本：`scripts/batch_backtest_runner.py`
- 回测数据源统一使用 `default`

---

## 2. 初始化输入模板

执行以下命令，一次性生成四个输入池模板：

```bash
python scripts/batch_backtest_runner.py --init-generator-templates
```

生成文件：
- `data/任务生成_策略池.csv`
- `data/任务生成_标的池.csv`
- `data/任务生成_区间池.csv`
- `data/任务生成_场景池.csv`

---

## 3. 数据准备说明

### 3.1 策略池（任务生成_策略池.csv）

字段：
- `策略ID`
- `是否启用`（1 启用，0 停用）

### 3.2 标的池（任务生成_标的池.csv）

字段：
- `股票代码`（如 `600000.SH`、`000001.SZ`）
- `市场标签`（主板/创业板/科创板）
- `行业标签`（银行/医药/半导体/消费等）
- `市值标签`（大盘/中盘/小盘）
- `是否启用`

建议：
- 覆盖多行业、多市值层级
- 不要只用单一股票

### 3.3 区间池（任务生成_区间池.csv）

字段：
- `区间ID`
- `开始日期`（YYYY-MM-DD）
- `结束日期`（YYYY-MM-DD）
- `行情标签`（牛市/熊市/震荡）
- `批次号`
- `是否启用`

建议：
- 三种行情都要有
- 每种行情建议准备多个区间

### 3.4 场景池（任务生成_场景池.csv）

字段：
- `场景标签`
- `成本档位`
- `滑点BP`
- `佣金费率`
- `印花税率`
- `最小手数`
- `是否T1`
- `K线周期`（建议至少含 `5min`、`1d`）
- `数据源`（必须为 `default`）
- `初始资金`
- `优先级`
- `最大重试`
- `是否启用`

## 4. 生成批量任务

### 4.1 覆盖式重建（推荐）

```bash
python scripts/batch_backtest_runner.py --generate-tasks --generate-mode replace
```

### 4.2 追加模式

```bash
python scripts/batch_backtest_runner.py --generate-tasks --generate-mode append
```

### 4.3 限制生成数量

```bash
python scripts/batch_backtest_runner.py --generate-tasks --generate-mode replace --generate-max-tasks 5000
```

默认输出任务文件：`data/批量回测任务.csv`

---

## 5. 覆盖率检查与硬门禁

## 5.1 只做覆盖率检查

```bash
python scripts/batch_backtest_runner.py --coverage-check
```

默认输出：`data/覆盖率检查结果.json`

### 5.2 启用硬门禁（推荐）

以下命令表示严格门禁：缺牛/熊/震荡样本、行业缺口、市值缺口均不允许：

```bash
python scripts/batch_backtest_runner.py --coverage-check --coverage-hard-gate --coverage-required-phases "牛市,熊市,震荡" --coverage-max-missing-phase-strategies 0 --coverage-max-missing-industries 0 --coverage-max-missing-sizes 0
```

门禁失败时：
- 脚本返回非0退出码（默认 2）
- 不进入回测执行阶段

可自定义失败返回码：

```bash
python scripts/batch_backtest_runner.py --coverage-check --coverage-hard-gate --coverage-fail-exit-code 9
```

---

## 6. 批量回测执行

### 6.1 串行执行

```bash
python scripts/batch_backtest_runner.py
```

### 6.2 多实例并发执行（真正大批量）

```bash
python scripts/batch_backtest_runner.py --parallel-workers 3 --base-urls "http://127.0.0.1:8000,http://127.0.0.1:8001,http://127.0.0.1:8002" --rate-limit-interval-seconds 0.5
```

说明：
- 单服务地址不建议并发，会相互影响
- 并发建议使用多服务实例（多端口）

---

## 7. 一条命令串起来

先生成任务，再跑硬门禁，最后继续执行回测：

```bash
python scripts/batch_backtest_runner.py --generate-tasks --generate-mode replace --coverage-check --coverage-hard-gate --run-after-coverage-check --coverage-required-phases "牛市,熊市,震荡" --coverage-max-missing-phase-strategies 0 --coverage-max-missing-industries 0 --coverage-max-missing-sizes 0
```
